# Веб‑мессенджер (клиент‑серверное веб‑приложение)

Клиент‑серверный веб‑мессенджер с REST API поверх HTTPS, аутентификацией и авторизацией.

## Цели и задачи
- Реализовать архитектуру «клиент‑сервер» 
- Обеспечить взаимодействие через REST API по протоколу HTTPS (самоподписанный сертификат).
- Настроить аутентификацию и авторизацию (JWT), персонализацию по аккаунту пользователя.
- Реализовать хранение данных на стороне бэкенда.
- Подготовить проект к демонстрации и последующему развитию.


## Структура проекта
```text
web-messenger/
├─ backend/           # Бэкенд
├─ frontend/          # Фронтенд
├─ certs/             # Сертификаты для HTTPS в режиме разработки
├─ docs/              # Материалы
└─ README.md          # Описание проекта и инструкция по запуску
```

## Основной функционал
- Регистрация и авторизация пользователей.
- Отправка и получение сообщений в общем чат‑канале.
- Привязка отправленных сообщений к учётной записи пользователя.
- Отображение истории переписки, обновление списка сообщений.

## Планы на будущее
- Перевод хранения данных на СУБД (SQLite/PostgreSQL).
- Поддержка общения в реальном времени (WebSocket).
- Расширение интерфейса (личные чаты, каналы, вложения).
